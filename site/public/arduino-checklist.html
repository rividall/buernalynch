Electronics Workshop

Keep checking for the common issues below!

Arduino plugged into USB
Green power LED lit on Arduino
Correct board selected
Tools → Board → Arduino Uno
Correct port selected
Tools → Port → (shows your Arduino)
Code compiled without errors
Orange text = errors to fix
Code uploaded successfully
"Done uploading" message appears
LED strip has power connected
Red to 5V, White to GND
LED strip data pin connected
Green to Pin 6 or your DATA_PIN
"Port not found" or "Upload failed"
  • Check USB cable is fully plugged in
  • Try a different USB port
  • Press reset button on Arduino
  • Go to Tools → Port and reselect
Orange text in console (compilation error)
  • Look for red underline in code
  • Check for missing semicolon ;
  • Check for missing bracket } or )
  • Did you close all quotes " " ?
LEDs don't light up
  • Check power: Red to 5V, White to GND
  • Check data: Green wire to correct pin
  • Is LED strip backwards? Look for arrows →
  • Try FastLED.setBrightness(100);
Sensor reads weird numbers
  • Check sensor wiring (VCC, GND, Signal)
  • Open Serial Monitor: Tools → Serial Monitor
  • Print value: Serial.println(value);
  • Adjust sensor position/lighting
"I broke it and don't know what I did!"
  • File → Open → Load your last saved version
  • Or re-copy the original snippet from folder
  • THIS IS WHY WE SAVE OFTEN! 💾
1 Save Your Working Code
Before making ANY changes: File → Save As
Give it a descriptive name like rainbow_v1.ino
✓ basic_working.ino
✓ sensor_test.ino
✓ final_day2.ino
2 Change One Thing at a Time
✅ Change brightness → Test → Save
❌ Change 10 things → Upload → "What broke?"
3 Comment Your Code
Comments help you remember WHY you did something!
// Changed from 100 - was too bright
int brightness = 50;
4 Test Frequently
After every small change:
• Upload code
• Watch what happens
• If it works: SAVE IT!
• If it breaks: Undo and try again
5 Read Error Messages
Orange text tells you WHAT is wrong and WHERE:
sketch.ino:23:5 = Line 23, position 5
